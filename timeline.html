    梦开始的地方

    2022年2月16日

    这是一个平凡的一天，Yanda 偶然在B站刷到了一条免费开服视频，Yanda 对此很感兴趣，于是便走上了一条不归路……

    与此同时，Yanda 的同学 Kongtou 得知了这个消息，他邀请了他在其他服务器结识的几个最好的朋友和一个同学，创建了一个服务器QQ群。

    服务器的一周目存档也在这一天被创建，版本为1.18.10。

    • 2022年2月16日
      Yanda 正在研究资源包，实现游戏内唱片音乐的更换。这是后来服务器唱片包和其他服务器资源包的基础。
    • 2022年2月17日
      服务器诞生了她的第一个公告，公告由 Yanda 撰写，记录了服务器的IP、端口、服务器上线人数、服务器配置（磁盘、内存、双核）等信息。
    • 2022年2月21日
      Yanda 开设了另外一个服务器，是一个娱乐性服务器，主要使用网上的玩法性存档（如起床战争、躲猫猫等）。这个行为也意味着服务器以后可能会成为一个群组服务器。
    • 2022年2月24日
      Yanda 之前在网上了解到了 LiteLoaderBDS，知道了插件能为服务器带来很多的可能性之后，顿感兴奋，并在今天为服务器新增了插件支持。
    • 2022年3月
      由于服主需要上学，平日无法打理服务器，又因为该免费面板服在长时间无在线人数的情况下会自动关闭，因此服务器经历了一段时间的冷淡期。

    萌芽的生长

    2022年4月

    Kongtou 当时也是其他服务器 MysteryGift （下称MG）的管理员，于是邀请了这个服务器的腐竹 Xiaoyou 和其他服务器的好友成鱼头到自己的服务器中，进行指导、宣传和担当技术支持。

    Kongtou 和 Yanda 也在这段时间逐渐了解了VPS、面板、菜单、行为日志、反作弊、备份、云端黑名单这些开服工具和必备插件。

    • 2022年4月17日
      Kongtou 决定为服务商付费，以使用能长期在线的服务器。此时服务器仍为面板服。
    • 2022年4月17日

      服务器的第一个兑换码横空出世。

      同时，Kongtou 提出了一个当 OP 的要求：和他关系好的基本都给（必须对服务器很熟或者开服元老）。
    • 2022年4月19日
      Kongtou 励志要成为全能腐竹。
    • 2022年4月20日
      服务器有了一个名称：Create top（下称CT），由成鱼头提出，群头像也在此时被设计出来，由成鱼头设计。
    • 2022年4月21日
      Kongtou 预算吃紧，但与 Xiaoyou 和其他管理进行一段时间的讨论后，被VPS的自由性和可操作性深深吸引，决定下个月倾家荡产也要给服务器更换VPS。
    • 2022年4月23日
      在 Xiaoyou 的帮助下，服务器有了第一个域名：mc-create.top，由 Xiaoyou 帮忙注册。
    • 2022年4月30日
      Kongtou 宣布娱乐服务器在节假日限时开放，进入方法为在主服跨服进入。这也是后来我们开设群组服的基础。

    2022年5月1日

    服务器由面板服更换为VPS，标志着 Create top 服务器正式成立。（远程桌面真的是太神奇了）

    • 2022年5月2日

      Yanda 设计了一个排版，用来展示服务器唱片包所用的音乐。

      往后的日子里，此唱片包也是在不断迭代更新之中。

    并不总是顺利

    • 2022年5月5日

      服务器发生了第一次熊服事件，危及规模较大，出生点附近的建筑群均被炸毁或烧毁。

      在 Xiaoyou 的帮助下，我们完善了日志和反作弊插件，并成功回档到了熊服的前一天。
    • 2022年5月18日

      MG 服主 Xiaoyou 宣布与CT解除一切合作关系。

      原因为我服一位管理在对方服务器进行破坏，并试图分裂玩家组织、破坏服务器玩家生态。

    立足、发展、不断探索

    • 2022年5月29日

      服务器管理 Yanda 创建了一个QQ频道。

      由于频道支持富文本发帖，Yanda 考虑今后在频道发布公告。
    • 2022年6月3日
      这天服务器来了一位新成员。他是服务器现存最老公会的领主 xbh6661800 。与此同时……
    • 2022年6月3日

      服主 Kongtou 正在收拾烂摊子，将之前退出的代理服主 Xiaoyou 极力挽回。

      后续 Xiaoyou 将涉事管理的账号送上云黑，此事件暂时平息。
    • 2022年6月7日

      FourWater，也是服务器后续管理的中坚力量，第一次进入服务器。

      由于他对新玩家非常友好、时不时给新玩家建议、时间充裕，且有非常积极的管理服务器意向，他后来被 Kongtou 无条件选为了我们服务器的管理员。
    • 2022年6月8日
      服主 Kongtou 宣布服务器将成立 Java 服务器分支，版本为1.12.2纯生存。
    • 2022年6月19日
      服务器管理员 Yanda 面临了一个困境：服务器因为插件适配无法升级至最新版本，因此 Yanda 考虑如何照顾与补偿玩家。
    • 2022年7月1日

      服务器技术员 Yanda 尝试将服务器更新至最新版本（从1.18更新至1.19），历时三小时，最终成功无损更新。

      这是 Yanda 第一次对服务端进行版本更新操作，Yanda 对本次升级感到非常兴奋。
    • 2022年7月17日

      CT服务器管理层处理了一起服务器管理失职事件。

      这是服务器第一次对管理员进行处罚，且如此严厉。涉事管理为服务器开服元老，也是服主的同学，该玩家现已离开服务器。
    • 2022年7月、8月
      随着东方智障（xbh6661800）、凯露酱（Axolotl3613）、润太祖（Honest man）、堀与宫村第一共和国主席（UsingUnicorn872）等一代活跃玩家在服务器的牵头活跃，和服务器广泛投放的宣传，服务器变得愈发热闹起来，新玩家越来越多，服务器设施越来越全面，存档也在持续膨胀。

    2022年8月

    服务器持续发布五月更新、六月更新、七月更新、八月更新，包括众多功能性插件和资源包的变化，服务器越来越完善，资源包提供的音频也越来越丰富。

    Yanda 也在着手准备服务器主城，叫做新手村，里面包含许多功能性建筑，供玩家自主处理一些事务。

    • 2022年8月12日
      服务器BOT第一次在群聊中发送消息，这是后续服务器自助白名单系统和群服互通的基础。
    • 2022年8月13日

      Create top 环境资源包实装，该资源包负责向服务器添加音频或修改游戏中的音频和部分材质。

      同期服务器的鸡叫被改为了坤叫。

    天塌了！

    2022年8月17日

    这是一个与平常无两样的小暑假。常日挂着远程桌面进行游戏的 Yanda 在悠闲地建设新手村时，突然收到远程桌面断开连接的消息，同时服务器也断开连接。重新连接很多次之后仍然连不上。

    意识到事情不妙的 Yanda 第一时间与服务商沟通，得知是服务商升级机器，但并未通知到我们，导致以前服务器的所有数据永久消失，且无法挽回。这无疑给了我们一个很大的打击。

    全体管理心如死灰，对服务商大失所望。整个后半月，服务器管理没有在群聊内发送过一条消息。（实际上是开学了）

    • 2022年8月22日
      Xiaoyou 在得知服务器数据全部被写 0 这件事后，好心地为CT服务器提供临时性服务器，等到CT的管理员回来之后，这些数据全都归CT所有。
    • 2022月8月24日
      一个服务器活跃玩家（堀与宫村第一共和国主席）公开吐槽服务器近月发生的事情，为自己这段时间在服务器的辛勤付出感到惋惜。
    • 2022年9月

      一个惊喜的发现！Yanda 偶然发现自己的电脑上还存放着5月份服务器在升级测试资源包时拷贝下来的服务器存档。

      在与 Kongtou 商量后，Yanda 决定在即将到来的一天半的中秋节假期中复活服务器（虽说插件全部要从0配置）。

    2022年9月10日

    经过一段时间的平息，管理层决定更换服务商（与MG服务器的服务商一致）重新开服。

    终于迎来了一次节假日，管理历时一晚上加一上午的鏖战，凭着本地测试拷贝的存档，和论坛上的插件资源，服务器基本成功复活，但存档从8月份回退到了5月份，众多活跃玩家数据无力挽回，新手村计划也无限期搁置。

    从此以后，服务器有了云端备份，Yanda 选用了当时不限速的阿里云盘作为存储媒介，不定期上传服务器存档。

    沉住气

    • 2022年9月11日
      其他子服务器（如娱乐服务器）也渐渐开始恢复。
    • 2022年9月28日
      服主 Kongtou 突然宣布自己退学，并宣称自己将有更多时间管理服务器。这或许正是他人格转变的开端。
    • 2022年10月
      管理员 FourWater 开始为服务器投放宣传，主要通过在各个频道发布帖子的方式。
    • 2022年11月12日
      服务器决定上线公会系统。彼时服务器涌现出许多公会团体，玩家活跃度也开始回暖。
    • 2022年11月29日
      因为服务器资源包，一个会长开始悄悄种下萌芽。又因为有了公会系统的辅助，服务器新晋公会《只因》开始迅速发展。

    友谊的结盟

    2023年1月7日

    服务器面临停服危机。起因为：在联系服务商续费时，服务商竟坐地起价，原先沟通好的一个月80突然变成了第一个月80，这让我们很是费解。由于我们的合作服务器 MG 和我们使用的是相同的服务商，据 Xiaoyou 说，此服务商还有过超开的迹象。

    最终两服达成共识，强强合作，毅然决定合并服务器，并更换其他有资质的服务商，存档数据共享，由此 MyCreateGift 服务器（下称MCG）成立。

    简要回顾 MG 服务器历史
    • 2023年1月7日
      面临停服危机，服务器群中的一个自建服务器的大佬主动提出为服务器贡献主机，Kongtou 封他为大股东。
    • 2023年1月8日
      经讨论，服主 Xiaoyou 宣布保留原 CT 的生存服务器存档，原CT服务器空岛将与原MG空岛耦合，且CT空岛将会停服。
    • 2023年1月9日
      服务器的 Logo 被重新设计。
    • 2023年1月12日
      MCG服务器新开设了一个大厅服，这是通往其他服务器的枢纽。
    • 2023年1月15日
      服务器管理层为庆祝 MG 服务器周年，在 MCG 服务器举办了周年庆活动，并诞生了服务器的第一个合照 —— 一周年合照。
    • 2023年1月22日
      MCG 服务器新年合照在MCG大厅服拍摄。
    • 2023年1月27日
      Kongtou 第一次在服务器群聊发送“测你们码”，且以“@全体成员”形式发送，引起群内成员广泛讨论。
    • 2023年1月28日
      一个新的服务器 —— 纷争势力服务器，计划在隔天开服，玩法类似于长期公会战。
    • 2023年1月29日
      纷争势力服务器还未开服，管理员 Kongtou 给一名玩家发放装备，然而却有人举报，事后这名被发装备的玩家在群内自爆，引起 Xiaoyou 注意。

    反骨

    2023年1月31日

    因服务器内部矛盾，原MG服主 Xiaoyou 与管理层爆发冲突。事件源于CT管理 Kongtou 因误操作引发的风波——其在未开放的纷争势力服务器中向玩家发放装备之后，该玩家却在交流群内"自爆"并暗示管理滥用权限，迫使 Kongtou 为平息事态将其移出群聊。然而Xiaoyou非但未调查真相，反而借机发难，意图剥夺 Kongtou 管理权限，其"护短"姿态与独断作风令团队深感忧虑。

    谈判中，以MG高管 Cheyi 为首的CT方代表，与 Z 等人据理力争，逐一揭露 Xiaoyou 架空原服主、篡改群名抹除CT印记、默许恶意举报等行径。面对"踩你脸上你还征求别人意见"的跋扈言论与"我授权给我自己"的荒诞操作，CT方始终秉持理性，以聊天记录为铁证，驳斥其"改名即换壳"的诡辩。当 Xiaoyou 最终以"我懒得"彻底摆烂时，CT方已完整收回服务器数据，宣告合作破裂。此役CT方以玩家利益为先、以程序正义为盾，在舆论与实质层面均赢得主动，为新服务器的独立运营奠定基石。
